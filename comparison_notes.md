# 四個價目表詳情頁對比分析

## 1. 客戶價目表 (rate-cards/[id])
- 基本資料區：開票公司、客戶、合約編號、服務類型、名稱、噸數、機種、**機號**、起點、終點、狀態
- 基本資料區：**沒有「日/夜」欄位**
- 費率區：日/夜 select + 費率+單位 + 中直費率 + OT費率+OT單位
- 唯讀顯示：日/夜卡片 + 費率卡片 + 中直卡片 + OT卡片（4格）
- 有效期區：生效日期、到期日期、**關聯工程項目**
- handleSave：backfill day_rate = rate, night_rate = 0
- 有 OT 時段費率表
- 有備註（僅編輯模式）

## 2. 租賃價目表 (fleet-rate-cards/[id])
- 基本資料區：開票公司、客戶、合約編號、服務類型、名稱、**日/夜 select**、噸數、機種、**機號**、起點、終點、狀態
- 費率區：費率+單位 + 中直費率 + OT費率（**沒有日/夜 select，因為放在基本資料區了**）
- 唯讀顯示：費率卡片 + 中直卡片 + OT卡片（3格，**沒有日/夜卡片**）
- 有效期區：生效日期、到期日期
- handleSave：不 backfill day_rate/night_rate
- 有 OT 時段費率表
- 有備註（僅編輯模式）

## 3. 供應商價目表 (subcon-rate-cards/[id])
- 基本資料區：開票公司、**供應商**、**車牌**、客戶、合約編號、服務類型、名稱、**日/夜 select**、噸數、機種、**機號**、起點、終點、狀態
- 費率區：費率+單位 + 中直費率 + OT費率 + **不包油 checkbox**（**沒有日/夜 select，因為放在基本資料區了**）
- 唯讀顯示：費率卡片 + 中直卡片 + OT卡片 + **包油卡片**（4格）
- 有效期區：生效日期、到期日期
- handleSave：不 backfill day_rate/night_rate
- 有 OT 時段費率表
- 有備註（僅編輯模式）

## 4. 租賃價目表-客戶端 (rental-rate-cards/[id])
- 基本資料區：開票公司、客戶、合約編號、服務類型、名稱、噸數、機種、起點、終點、狀態
- 基本資料區：**沒有「日/夜」欄位**、**沒有機號**
- 費率區：**日/夜 select** + 費率+單位 + 中直費率 + OT費率
- 唯讀顯示：費率(日/夜)卡片 + 中直卡片 + OT卡片（3格）
- 有效期區：生效日期、到期日期
- handleSave：**不 backfill day_rate/night_rate**（但客戶價目表有 backfill）
- 有 OT 時段費率表
- 有備註（僅編輯模式）

## 發現的不一致問題

### 問題 1：「日/夜」欄位位置不一致
- 客戶價目表：放在**費率區**
- 租賃價目表：放在**基本資料區**
- 供應商價目表：放在**基本資料區**
- 租賃-客戶端：放在**費率區**
→ 應該統一放在**基本資料區**（因為日/夜是這筆價目的屬性，不是費率的一部分）

### 問題 2：「日/夜」唯讀顯示不一致
- 客戶價目表：費率區有獨立的灰色「日/夜」卡片
- 租賃價目表：基本資料區有文字顯示，費率區沒有
- 供應商價目表：基本資料區有文字顯示，費率區沒有
- 租賃-客戶端：費率區的費率卡片標題帶 `(日/夜)` 後綴
→ 應該統一：基本資料區顯示文字，費率區不重複

### 問題 3：handleSave 中 backfill day_rate 不一致
- 客戶價目表：有 `day_rate = rate, night_rate = 0`
- 其他三個：沒有 backfill
→ 應該統一都做 backfill（向後兼容）

### 問題 4：缺少機號欄位
- 客戶價目表：有機號
- 租賃價目表：有機號
- 供應商價目表：有機號 + 車牌
- 租賃-客戶端：**沒有機號**
→ 租賃-客戶端應該加上機號

### 問題 5：OT 費率單位選擇不一致
- 客戶價目表：OT 費率有單位選擇（ot_unit）
- 其他三個：OT 費率沒有單位選擇
→ 應該統一都有或都沒有
